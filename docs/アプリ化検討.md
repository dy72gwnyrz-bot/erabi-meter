# スマホアプリ化(iOS / Android)検討メモ

2026-07-10 時点の検討結果。現状は PWA(HTML/CSS/JS 1ファイル + Service Worker、データは localStorage のみ、サーバーなし)。

## 選択肢の比較

| 方式 | iOS | Android | 工数 | 費用 | 備考 |
|---|---|---|---|---|---|
| A. PWA のまま(現状) | ホーム画面追加のみ | ホーム画面追加/インストールバナー | ゼロ | 無料 | ストア配布不可 |
| B. TWA(Bubblewrap) | 不可 | **Play ストア配布可** | 極小(数時間) | Play 登録 $25(買い切り) | PWA をそのまま包む公式ルート |
| C. Capacitor ラップ | **App Store 配布可** | Play ストア配布可 | 小(1〜3日) | Apple $99/年 + Play $25 | 既存 index.html をほぼ無改造で WebView 化 |
| D. React Native / Flutter 書き直し | 可 | 可 | 大(数週間) | 同上 | このアプリ規模では過剰。不採用 |

## 推奨プラン

**段階的に B → C。いきなり D はやらない。**

1. **Android: TWA(Trusted Web Activity)を Bubblewrap で生成** — 最有力
   - すでに GitHub Pages で HTTPS 配信済みなので前提条件はクリア済み。`npx @bubblewrap/cli init --manifest https://dy72gwnyrz-bot.github.io/erabi-meter/manifest.webmanifest` でほぼ自動生成。
   - アプリの実体は Pages 上の PWA なので、**Web 側を更新すればアプリも即更新される**(ストア再審査不要)。
   - 必要作業: assetlinks.json の設置(`.well-known/`)、署名鍵の生成、Play Console 登録($25)。
2. **iOS: 当面は「ホーム画面に追加」運用を継続**
   - standalone 表示・オフライン動作は既に達成済みで、体験はほぼアプリ。
   - App Store に出したくなったら **Capacitor** でラップ。既存コードがフレームワーク非依存の素の JS なので相性が良い。
   - 注意: Apple 審査ガイドライン 4.2(minimum functionality)で「ただの Web ラッパー」はリジェクトされ得る。オフライン完結・ローカルデータという性質は説明材料になるが、個人利用なら TestFlight 配布(審査が緩い)で十分という判断もあり。
   - Mac + Xcode が必須。Apple Developer Program $99/年。

## アプリ化の前にやるべき技術課題

- **localStorage からの移行**: iOS Safari は「最近使っていないサイト」のスクリプト書き込みデータ(localStorage 含む)を7日で削除することがある(ホーム画面追加アプリは対象外とされるが保証はない)。アプリ化する・しないに関わらず、
  1. `navigator.storage.persist()` の要求(1行で追加可能)
  2. **結果のエクスポート/インポート機能**(JSON をクリップボード or ファイルへ)
  を先に入れるとデータ消失リスクと端末間移行の両方が解決する。Capacitor 化する場合は Preferences プラグインへの移行が確実。
- **端末間のデータ同期**: 本格的にやるならバックエンドが必要になり「サーバー不要・外部送信なし」の設計思想と衝突する。エクスポート/インポートで足りるかを先に見極めるべき。
- **アイコン**: iOS はマスカブルアイコンの余白要件が厳しめ。現行 icon-512.png の余白確認を。

## 結論

- **コスパ最良は「Android=TWA、iOS=現状PWA継続」**。追加開発ほぼゼロで Play ストア配布まで到達できる。
- App Store 配布が必要になった時点で Capacitor を導入(既存コード資産はそのまま活きる)。
- どのルートでも先にエクスポート/インポートと `storage.persist()` を実装しておくのが安全。
